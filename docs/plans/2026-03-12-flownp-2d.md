# FlowNP_2D Pure 2D Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Remove pocket and 3D runtime dependencies from `FlowNP_2D` and make `QM9` and `COCONUT` support end-to-end 2D preprocessing, loading, training, and sampling.

**Architecture:** Converge the repository onto a single active runtime path: preprocessing scripts generate training-compatible artifacts, `MoleculeDataset` loads only categorical 2D features, `FlowMol` uses `CTMCVectorField2D`, and sampling exports molecules by generating RDKit 2D coordinates when needed. All active pocket and 3D feature paths are removed from the main import graph instead of being left partially disabled.

**Tech Stack:** Python, PyTorch, PyTorch Lightning, DGL, RDKit, pytest, YAML configs
